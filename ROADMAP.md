# Documentation Roadmap

Single authoritative tracker for **every document this project produces or has planned** вҖ” what
exists, what depends on what, and what's still open. Humans and future sessions read this file
first to answer "what's done, what's in flight, what's left" without re-deriving it from `git
log` or scanning the tree.

**Update this file whenever a document's status changes.** Finish a doc вҶ’ flip it to вң…. Start
one вҶ’ рҹҡ§. Identify a new doc вҶ’ add a row with `вӣ” Planned`. Don't let this file drift from
reality. The owning skill flips its rows together with the theme `INDEX.md` in the same commit.

## Status legend

| Symbol | Meaning |
|---|---|
| вң… | Done вҖ” written, current, matches the implementation/research it describes. |
| рҹҡ§ | In progress вҖ” drafted but incomplete, or gate not fully closed. |
| вӣ” | Planned вҖ” identified as needed, not yet started. |
| вҷ»пёҸ | Living вҖ” intentionally never "done"; updated continuously. |

`Depends on` lists the IDs whose content a document assumes or must stay consistent with вҖ” not a
build order.
