- The explicit assumptions [MSTR-001](../master/MSTR-001-program-vision.md) rests on. Each row
  carries a **trigger**: the observable condition under which the assumption stops holding. A
  fired trigger is not silently absorbed — the row returns to `00-pipeline-manager` for
  re-triage, and any resulting vision change runs through `01-vision` with its blast radius
  enumerated.

| A5 | **Audience is casual/all-ages short-session play**; content stays family-friendly and non-violent; no fail states — expected to hold even as the world scales up per C7 (bigger ≠ harder/longer). *(Bootstrap-derived — the shipped game implies but nowhere states this; recorded here rather than blocking on a user interview.)* **v3.0 note:** C9's item-agnostic, per-region key-item theming and C10's world-scale parameter both inherit this assumption unchanged — a larger generated world or a re-themed collectible must not become longer/harder in the session-shape sense, only bigger in the world sense. **Trigger fired, v4.0 (2026-07-17):** the project owner directed a bounded audience/tone carve-out — an opt-in combat sub-mode for a parent/adult player, layered on Infinite Mode, gated behind its own explicit entry point (`MSTR-001` C11, `BL-0133`). **This assumption is re-scoped, not retired**: it still holds in full for every mode a child reaches through the game's own default flow (finite mode; Infinite Mode's base collect-loop) — C11's carve-out applies only inside its own gated sub-mode, which downstream design (`03`→`04`→`06`) must keep genuinely separate from that default flow, not blend into it. | The project owner directs a *further* tonal, difficulty, or audience shift (e.g. extending C11's carve-out into the base game, or reversing it). | MSTR-001 §2/§3 C6/C9/**C11**, R201/R206 research, every content FS, `ADS-002` |
| A6 | **The shipped Bunny Quest behavior is the intended design** (9 zones, 9-carrot win condition, star/flower scoring, auto-load). **Revised (2026-07-06):** the original wording ("v2.1," BL-0001/BL-0003) described the *pre-rewrite* game and is retracted — those two backlog entries need re-verification against the current code before being treated as still-valid bug reports (see MSTR-001 §8). **Revised again (v3.0, 2026-07-09):** this assumption now explicitly does **not** extend to the fixed 3×3 layout or auto-load-on-boot going forward — both are named, deliberate C5 amendments in flight (C9/C10, C2), not contradictions of this assumption. | A7 | **English-only text** on the existing Latin font tiles. | A localization request, or text content exceeding the font tile budget. | GDS-08, `tiles.py` font range, content FS specs |
| A8 | **The working docs/tests will be brought back into sync with shipped code** before requirements-baselining (stage 04) relies on them. **Resolved (2026-07-07):** `test_rom.py` rewritten and independently verified (`IP-9010`/`VR-9010`, 125/125); `Claude.md`/`memory.md`/`README.md` refreshed (`IP-9030`, 2026-07-09, awaiting independent verification). | A9 *(new, v3.0)* | **A generated world's only entropy sources are its seed and world-scale parameter** — no dependence on the DIV register, uninitialized RAM, timing, or any other non-reproducible input anywhere in the generation algorithm. | A10 *(new, v3.0)* | **No text/dialogue rendering engine is needed** — C9's visual narrative is delivered entirely through world structure (biome adjacency, item theming), reusing the existing font-tile digit/glyph rendering only for menu and HUD text, never narrative prose. **Retired assumptions:** none. Retire by striking the row's status, never by deleting the row.
